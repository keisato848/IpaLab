# プロジェクトマネージャ試験対策プラットフォーム「PM Exam DX」要件定義書W

## 1. プロジェクト概要
### 1.1 目的
既存の過去問演習サイトの課題（UIの断片化、記述・論述対策の属人性）を解決し、学習心理学（忘却曲線）とデータ分析を組み合わせることで、午前Iから午後IIまで、最短ルートでの合格を支援する。

### 1.2 ターゲットユーザー
- プロジェクトマネージャ試験（PM）合格を目指すITエンジニア。
- **午前I試験からの受験が必要な方、および免除期限が迫っている方。**
- 隙間時間を活用して効率的に学習したい社会人。
- 午後II（論文）の書き方や構成に不安を感じている受験者。

## 3. 機能要件

### 3.1 午前I・II：効率化と定着の最大化
- **Spaced Repetition (忘却曲線) エンジン:**
    - 午前I（広範囲）および午前II（専門）の過去問に対し、正誤履歴から適切な復習タイミングを提示。
- **午前I免除管理機能:**
    - 免除資格の有無と有効期限を登録。期限切れを防止するアラート通知。
- **進捗の可視化 (Burn-down Chart / EVMS):**
    - 試験日をゴールとした「残り学習量（午前I・II、午後I合計）」を可視化。
    - 学習進捗をEVMS（アーンドバリュー）の観点で表示。
- **ゲーミフィケーション:**
    - 連続学習ストリーク、分野別ランク、合格バッジ付与。

### 3.2 午後I・II：記述・論述対策のDX
- **根拠リンク視覚化（午後I）:**
    - 解答と問題本文中の「根拠となる一文」を紐付けてハイライト表示。
- **論述ナビゲーター（午後II）:**
    - **モジュール式入力:** スマホから1章ずつ（800字程度）を隙間時間に作成・保存。
    - **オートセーブ:** 入力中のデータ損失を防ぐため、端末ローカルおよびクラウドへ自動保存。
- **「合格者プロット」の共有:**
    - 合格者が作成した論文の骨子（章立て）をテンプレートとして閲覧・参考にできる機能。

### 3.3 パーソナライズ・AI機能（拡張フェーズ：有料プラン想定）
- **IPA採点基準シミュレーター:**
    - 記述・論文に対し、採点講評を学習したAIが評価（A〜D判定）。
- **弱点クラスター分析:**
    - 誤答傾向を「知識不足」「読解ミス」「時間不足」に分類。

## 4. UI/UX 戦略
### 4.1 デザインコンセプト
- **テーマ:** Digital Ocean（深い紺色と明るい水色のコントラスト）。
- **効果:** 青色による集中力の維持。無駄を削ぎ落としたミニマルデザイン。

### 4.2 主要画面設計
- **統合ダッシュボード:** 午前I・IIの正答率、バーンダウンチャート、今日のタスク。
- **フォーカス・エディタ:** 不要なメニューを隠し、問題文と解答欄のみに集中できる論文作成UI。
- **スマホ専用暗記カード:** 午前Iの広範な用語、論文の「決め台詞」を隙間時間に学習。

## 5. 既存サイト（pm-siken.com等）に対する優位性
| 比較項目 | 既存サイト | 本プラットフォーム（PM Exam DX） |
| :--- | :--- | :--- |
| **試験カバー範囲** | 各試験区分で独立 | **午前I〜午後IIまで統合管理（免除管理含む）** |
| **進捗管理** | 自己管理（表形式が主） | **バーンダウンチャートによる直感的な管理** |
| **学習継続率** | 受動的 | **Spaced Repetitionによる能動的介入** |
| **午後II対策** | 模範解答の閲覧のみ | **合格者プロットの共有・モジュール式作成** |
| **モバイル体験** | PCサイトのレスポンシブ | **ネイティブアプリによるオートセーブと操作性** |

## 6. 出典および参考文献
- **過去問題・公式解答:**
    - 独立行政法人情報処理推進機構 (IPA)「情報処理技術者試験 過去問題」
    - **権利表記:** 各問題ページおよびヘルプ内に「出典：IPA 情報処理技術者試験 過去問題」を明記。
- **学習アルゴリズム:**
    - Pimsleur, Anki 等で採用されている拡張型忘却曲線アルゴリズム。

---

## 7. 非機能要件
- **スケーラビリティ:** 午前Iが含まれることでユーザー層が拡大するため、Cosmos DBのオートスケール設定を最適化。
- **可用性:** 99.9%以上の稼働率を目標。
- **データ保護:** ユーザーの作成した論文データ（機密性の高い実務エピソード等）はAzure Blob Storageに暗号化して保存。

## Appendix：次のステップへのアドバイス

1. **午前Iデータの効率的な収集:**
   - 午前Iは高度情報処理試験共通のため、他区分（ST, AU等）の受験者もターゲットにできる可能性があります。将来的な横展開を見据えたDB構造（試験区分フラグの保持）を推奨します。
2. **免除ロジックの実装:**
   - 「合格から2年間免除」というルールを正確に判定し、ユーザーに再受験の必要性を早期に気づかせるUI/UXが差別化ポイントになります。
3. **コンテンツのボリューム管理:**
   - 午前Iは出題範囲が非常に広いため、全問網羅するよりも「頻出問題」に絞ったレコメンド機能を優先すると、ユーザーの学習効率が上がります。
4. **段階的リリースの検討:**
   - MVP（最小機能製品）として「午前I・II対策＋バーンダウンチャート」を先行リリースし、論文サポート機能は後追いで追加する開発サイクルが現実的です。
